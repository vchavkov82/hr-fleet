2. Create `.github/workflows/ci.yml`:
   - Trigger: push to main, pull_request to main
   - Single job `ci` on `ubuntu-latest`
   - Steps:
     a. checkout with `fetch-depth: 2` (for Turborepo affected detection)
     b. `pnpm/action-setup@v3` with `version: 9`
     c. `actions/setup-node@v4` with `node-version: 20`, `cache: pnpm`

     f. `pnpm turbo run lint --affected`
     g. `pnpm turbo run typecheck --affected`
     h. `pnpm turbo run test --affected`
     i. `pnpm turbo run build --affected`

<task type="auto">
  <name>Task 2: Create production deploy workflow</name>
  <files>.github/workflows/deploy-prod.yml</files>
  <action>
1. Create `.github/workflows/deploy-prod.yml`:
   - Trigger: push to main (only after CI passes — use `workflow_run` or separate job with `needs`)
   - Approach: SSH to VPS, pull latest code, rebuild with Docker Compose
   - Use `appleboy/ssh-action@v1` for SSH deployment:
```yaml
name: Deploy Production
on:
  push:
    branches: [main]
jobs:
  ci:
    uses: ./.github/workflows/ci.yml
  deploy:
    runs-on: ubuntu-latest
    needs: [ci]
    if: github.ref == 'refs/heads/main'
    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_SSH_KEY }}
          script: |
            cd /opt/hr
            git pull origin main
            docker compose -f deploy/podman-compose.yml pull
            docker compose -f deploy/podman-compose.yml up -d --build
```

   NOTE: If `workflow_run` approach is cleaner (CI as reusable workflow), use that instead. The key requirement is: deploy ONLY runs after CI passes. Two environments only: dev (local) + production. No staging.

2. Verify YAML syntax: ensure all workflow files parse correctly.
