<task type="auto">
  <name>Task 2: Webhook registration and async delivery</name>
  <files>services/api/internal/handler/webhook.go, services/api/internal/service/webhook.go, services/api/internal/worker/webhook.go</files>
  <action>
1. Create `internal/service/webhook.go`:
   - `WebhookService` struct with db, asynqClient
   - `Register(ctx, url, events[]string, createdBy) (WebhookRegistration, secret, error)` — generate HMAC secret, store
   - `List(ctx) ([]WebhookRegistration, error)` — mask secrets in response
   - `Deactivate(ctx, webhookID) error`
   - `Dispatch(ctx, event string, payload any) error` — find active registrations matching event, enqueue Asynq delivery tasks
   - Events: `payroll.completed`, `payroll.failed`, `employee.created`, `employee.updated`, `leave.approved`, `leave.rejected`

2. Create `internal/handler/webhook.go`:
   - `POST /api/v1/webhooks` — register webhook (RBAC: ManageWebhooks), returns secret ONCE
   - `GET /api/v1/webhooks` — list registrations (secrets masked)
   - `DELETE /api/v1/webhooks/{id}` — deactivate
   - `GET /api/v1/webhooks/{id}/deliveries` — list delivery history

3. Create `internal/worker/webhook.go`:
   - Asynq task type: `webhook:deliver`
   - `DeliverWebhookTask(ctx, task) error`:
     - POST to webhook URL with JSON payload
     - Add `X-Webhook-Signature` header: HMAC-SHA256 of payload using webhook secret
     - Record response code and any error
     - Retry up to 5 times with exponential backoff (Asynq handles retry config)
     - Mark delivery as succeeded/failed based on 2xx response
  </action>
