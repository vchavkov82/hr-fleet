  <name>Task 2: Payroll run lifecycle, payslip endpoints, and Asynq worker</name>
  <files>services/api/internal/handler/payroll.go, services/api/internal/service/payroll.go, services/api/internal/worker/payroll.go, services/api/internal/handler/payslip.go, services/api/internal/service/payslip.go, services/api/platform/odoo/payslip.go</files>
  <action>
1. Create `internal/service/payroll.go`:
   - `PayrollService` struct with db, odoo, asynqClient, taxCalculator
   - State machine: validTransitions map (from RESEARCH.md)
   - `Create(ctx, periodStart, periodEnd, createdBy) (PayrollRun, error)` — creates draft
   - `Approve(ctx, runID, approvedBy) error` — draft->approved transition, RBAC: ApprovePayroll
   - `TriggerProcessing(ctx, runID) error` — approved->processing, enqueues Asynq task, returns immediately
   - `GetStatus(ctx, runID) (PayrollRun, error)` — for polling
   - `transitionStatus(ctx, runID, from, to) error` — validates transition

2. Create `internal/worker/payroll.go`:
   - Asynq task type: `payroll:process`
   - `ProcessPayrollTask(ctx, task) error`:
     a. Get payroll run from DB
     b. Fetch all active employees from Odoo
     c. For each employee: get gross salary, call tax.Calculate(), create payslip record
     d. If ANY employee fails: mark run as failed with error_details JSONB, return error (fail entire batch per discretion)
     e. If all succeed: mark run as completed, set completed_at
   - Completed runs are immutable (no update/delete endpoints)

3. Create `internal/handler/payroll.go`:
   - `POST /api/v1/payroll-runs` — create draft (RBAC: ManagePayroll)
   - `POST /api/v1/payroll-runs/{id}/approve` — approve (RBAC: ApprovePayroll)
   - `POST /api/v1/payroll-runs/{id}/process` — trigger async, return 202 with `{status_url: "/api/v1/payroll-runs/{id}"}`
   - `GET /api/v1/payroll-runs/{id}` — get status (RBAC: ViewPayroll)
   - `GET /api/v1/payroll-runs?status=&page=&per_page=` — list runs

4. Create `internal/handler/payslip.go` + `internal/service/payslip.go`:
   - `GET /api/v1/payslips?payroll_run_id=&employee_odoo_id=` — list payslips
   - `GET /api/v1/payslips/{id}` — get payslip detail with calculation breakdown
   - `POST /api/v1/payslips/{id}/confirm` — confirm payslip (mark as final)

5. Create `platform/odoo/payslip.go`:
   - `SearchPayslips(ctx, filters, offset, limit)` — hr.payslip search_read (for reading Odoo-side payslip data if needed)
