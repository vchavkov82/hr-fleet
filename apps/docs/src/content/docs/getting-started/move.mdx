## When you should use this guide

Use this guide when you need to move:

1. A documentation page to a new location
2. Images/assets referenced by docs
3. An entire section (folder) of docs

## Moving a documentation page

1. Move the file to its new folder under `src/content/docs/`.
2. Update internal links that referenced the old path.
3. Update any imports at the top of the page (for example components or assets).
4. Check the sidebar ordering via frontmatter `sidebar.order`.

:::tip
Prefer relative links that match the final URL structure (the path under `src/content/docs`).
:::

## Moving assets (images, files)

1. Move the asset under `src/assets/`.
2. Update all references to the asset in MDX/Markdoc pages.
3. Verify the asset still renders in the page after the move.

## Moving a whole section

1. Move the folder under `src/content/docs/`.
2. Update links in other sections that point to pages inside the moved folder.
3. If the section is included via `autogenerate` in `astro.config.mjs`, ensure the new directory name matches the configured directory.

## Verification checklist

1. Run the docs dev server and open the moved page.
2. Ensure there are no broken links.
3. Ensure the page appears in the sidebar where you expect it.
