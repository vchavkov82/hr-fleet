## Security Overview

The HR platform handles sensitive personal and financial data, making security a foundational concern. The platform implements defense-in-depth with multiple layers of protection: network-level isolation through container networking, application-level authentication and authorization, data-level encryption and audit logging, and operational security through secret management.

### Security Layers

- **Authentication** — All API endpoints (except health checks and public routes) require either a valid JWT bearer token or a scoped API key. JWTs use RS256 asymmetric signing with HS256 fallback.
- **Authorization** — A seven-role RBAC system with eleven granular permissions controls access to every resource. Roles are tenant-scoped, preventing cross-organization access.
- **Data protection** — Monetary values stored as integers (stotinki) to prevent rounding errors. All sensitive operations are audit-logged with actor, action, and timestamp.
- **API security** — Rate limiting on authentication endpoints, HMAC-SHA256 signed webhooks, and scoped API keys with configurable permissions and expiration.

### Compliance Considerations

The platform is designed with Bulgarian data protection requirements in mind. Employee personal data is isolated per tenant, and the audit log provides a complete trail of who accessed or modified sensitive information. All API communication uses HTTPS through the Caddy reverse proxy, which handles automatic TLS certificate provisioning.

See the authentication guide for detailed information on JWT tokens, API keys, and the RBAC permission model.
