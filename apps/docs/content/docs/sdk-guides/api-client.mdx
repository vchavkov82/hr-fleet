## Authentication

The API supports two authentication methods: JWT bearer tokens and API keys. Use JWT tokens for user-facing applications and API keys for server-to-server integrations.

### Obtaining a JWT Token

```bash
curl -X POST https://api.jobshr.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@example.com", "password": "your-password"}'
```

The response includes an `access_token` (15-minute expiry) and a `refresh_token` (7-day expiry). Include the access token in subsequent requests via the `Authorization: Bearer <token>` header.

### Using API Keys

Generate an API key from the admin panel under Settings. API keys are scoped to specific permissions and passed via the `X-API-Key` header.

```bash
curl https://api.jobshr.com/api/v1/employees \
  -H "X-API-Key: hrk_your_api_key_here"
```

## Employee Operations

### List Employees

```bash
curl https://api.jobshr.com/api/v1/employees?limit=20&offset=0 \
  -H "Authorization: Bearer <token>"
```

### Create an Employee

```bash
curl -X POST https://api.jobshr.com/api/v1/employees \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Ivan",
    "last_name": "Petrov",
    "email": "ivan.petrov@company.bg",
    "department": "Engineering",
    "position": "Software Developer",
    "employee_type": "employee"
  }'
```

## Pagination

All list endpoints return paginated results with the following structure:

```json
{
  "data": [],
  "total": 142,
  "limit": 20,
  "offset": 0
}
```

Use `offset` and `limit` query parameters to navigate through results. The `total` field indicates the complete count of matching records.

## Error Handling

The API returns standard HTTP status codes. Error responses include a consistent JSON body:

```json
{
  "error": "validation_error",
  "message": "Email is required",
  "details": {"field": "email"}
}
```

Common status codes: 400 (validation), 401 (unauthenticated), 403 (forbidden), 404 (not found), 429 (rate limited), 500 (server error).
