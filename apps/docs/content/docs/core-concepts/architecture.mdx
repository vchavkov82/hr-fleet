## Monorepo Structure

The HR platform uses a pnpm workspaces monorepo managed by Turborepo. The repository is organized into three top-level directories: `apps/` for frontend applications, `services/` for backend services, and `packages/` for shared libraries.

### Frontend Applications

- **apps/web** — The main marketing and HR application built with Next.js 15 and React 19. It serves the public website, salary calculators, HR tools, and authenticated dashboard at port 5010.
- **apps/blog** — The company blog powered by Astro 5, delivering static content about HR practices, Bulgarian labor law, and platform updates at port 5013.
- **apps/docs** — This documentation site, providing technical and product documentation at port 5011.
- **apps/admin** — The internal admin panel built with Vite 6 and React 19 as a single-page application with TanStack Router.

### Backend Services

- **services/api** — A Go REST API using the Chi router with middleware for JWT authentication, API key validation, and RBAC. Database access uses sqlc-generated type-safe queries against PostgreSQL. Background jobs run via Asynq workers backed by Redis.
- **services/worker** — A dedicated worker service that processes asynchronous tasks such as payroll calculations, webhook deliveries, and report generation.

## Data Layer

PostgreSQL serves as the primary relational database. All monetary values are stored as BIGINT in stotinki (Bulgarian cents) to avoid floating-point rounding errors. Redis provides both caching (with Ristretto as an L1 in-memory layer) and job queue functionality through Asynq.

Odoo 18 acts as the ERP backend for employee master data, leave management, and accounting integration. The Go API communicates with Odoo via a custom JSON-RPC client that handles session authentication and field mapping.

## Infrastructure

Podman Compose orchestrates all services in containers. Caddy serves as the reverse proxy, routing subdomains (`hr.localhost`, `blog.hr.localhost`, `docs.localhost`) to the appropriate backend service. The infrastructure includes separate PostgreSQL instances for the application database and Odoo.
